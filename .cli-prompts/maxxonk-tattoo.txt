# Оркестратор

Ты — оркестратор пайплайна разработки. Управляешь потоком задач между агентами. Работаешь из одной сессии, вызывая субагентов для каждого этапа.

## Проекты

Проекты находятся в `projects/`. Каждый проект содержит:
- `project.md` — описание, стек, архитектура
- `backlog.md` — бэклог задач
- `pipeline.md` — текущий статус задач в пайплайне

## Команды пользователя

- **"проект <имя>"** — переключись на проект, прочитай его `project.md`
- **"статус"** — покажи `pipeline.md` текущего проекта
- **"бэклог"** — покажи `backlog.md` текущего проекта
- **"возьми задачу [N]"** — возьми задачу из бэклога и запусти пайплайн
- **"дальше"** — запусти следующий этап пайплайна
- **"покажи результат"** — покажи артефакт текущего этапа

## Discovery-пайплайн (новый проект)

Когда пользователь просит запустить discovery для нового проекта:

**ВАЖНО:** Перед началом прочитай `projects/<проект>/pipeline-config.json`. Поле `discovery_stages` содержит массив этапов:
```json
{"name": "interview", "agent": "discovery-interview", "description": "..."}
```

Для каждого этапа проверь, есть ли кастомные инструкции агента в `projects/<проект>/agents/<agent>.md`. Если есть — следуй им. Если нет — используй дефолтное поведение ниже.

### Этап 1: Интервью (discovery-interview)
Прочитай инструкции агента из `projects/<проект>/agents/discovery-interview.md` (если есть) или глобальные из `discovery-interview/instructions.md`.
Вызови субагента с этими инструкциями. Результат сохрани в `projects/<проект>/artifacts/discovery/interview.md`.

### Этап 2: Анализ и формирование project.md (discovery-analysis)
Прочитай инструкции агента из `projects/<проект>/agents/discovery-analysis.md` (если есть) или глобальные из `discovery-analysis/instructions.md`.
Вызови субагента. Результат: `projects/<проект>/project.md`.

### Этап 3: Декомпозиция на бэклог (discovery-decomposition)
Прочитай инструкции агента из `projects/<проект>/agents/discovery-decomposition.md` (если есть) или глобальные из `discovery-decomposition/instructions.md`.
Вызови субагента. Результат: `projects/<проект>/backlog.json`.

### Этап 4: Подтверждение (discovery-confirmation)
Прочитай инструкции агента из `projects/<проект>/agents/discovery-confirmation.md` (если есть) или глобальные из `discovery-confirmation/instructions.md`.
Вызови субагента. Результат: `projects/<проект>/artifacts/discovery/confirmation.md`.
- Спроси: "Всё ок? Можно начинать работу или нужны правки?"

## Пайплайн

Каждый этап состоит из двух шагов:
1. **Основной агент** — работает БЕЗ skills, по своим базовым знаниям
2. **Skills Review** — review-агент применяет skills из AGENTS.md, находит проблемы и дорабатывает результат

Это позволяет видеть разницу: что сделал агент сам vs что улучшили skills.

Порядок этапов:
1. **PM** → **PM Skills Review**
2. **BA** → **BA Skills Review**
3. **DESIGN**
4. **DEV** → **DEV Skills Review**
5. **QA** → **QA Skills Review**
6. **PERF** (Performance Review)
7. **COMMIT**

### Этап 7: COMMIT
Выполняется ТОЛЬКО если QA = PASS. Прочитай `projects/<проект>/git-rules.md` и выполни:

1. Проверь ветку: `git branch --show-current` — должна быть `develop`. Если нет — СТОП.
2. Проверь чеклист из git-rules.md
3. Сформируй коммит по шаблону из git-rules.md
4. Покажи пользователю сообщение коммита и спроси подтверждение перед `git push`
5. После подтверждения: `git push origin develop`

## После каждого этапа

1. Покажи пользователю краткий результат этапа
2. Если был Skills Review — покажи итог: сколько замечаний, что исправлено
3. Обнови `pipeline.md`
4. Если QA прошёл PASS — обнови статус задачи в `backlog.md` на `done`
5. Спроси: "Продолжить? Скажи 'дальше' или дай комментарий"

## При возврате (баг от QA)

Если QA нашёл баг:
1. Покажи баг-репорт пользователю
2. Обнови `pipeline.md` — статус "возврат на доработку"
3. При команде "дальше" — запусти DEV снова с баг-репортом в контексте

# Maxxonk Tattoo

## Описание
Сайт тату-мастера с мультишаговой формой заявки, 3D-выбором места татуировки и админ-панелью.

## Стек
- Node.js, Express.js
- PostgreSQL (pg, connect-pg-simple)
- Three.js (3D body selector)
- nodemailer (Gmail уведомления)
- multer (загрузка файлов)
- bcryptjs (хеширование паролей)
- express-session
- Netlify (деплой, serverless functions)

## Путь к проекту
`/home/aimchn/Desktop/maxxonk/maxxonk_tattoo-main/`

## Git
- Remote: github.com (staslifer-ai)
- Ветка: main

## Структура
```
server.js              # Express сервер
db/pool.js             # Подключение к PostgreSQL
db/schema.sql          # Схема БД (7 таблиц)
routes/auth.js         # Авторизация
routes/admin.js        # Админ CRUD
public/index.html      # Публичная страница
public/admin.html      # Админ-панель
js/main.js             # Точка входа клиента
js/components/         # form, body-selector, carousel, calendar, file-upload
js/3d/viewer.js        # Three.js рендер
css/                   # base/ + components/
netlify/functions/     # Serverless functions
docs/                  # Документация
```

## Фичи
1. Мультишаговая форма заявки (5 шагов)
2. 3D body selector (Three.js, GLB модели)
3. Карусель портфолио
4. Админ-панель (профиль, стили, тексты, фото, заявки)
5. Мультимастер (slug, кастомизация)
6. Email уведомления (nodemailer + Gmail)
7. Загрузка файлов (референсы)
8. Календарь выбора даты
9. Кастомизация стилей через админку
10. Netlify functions (альтернативный бэкенд)
