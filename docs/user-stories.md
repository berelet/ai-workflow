# User Stories: Рефакторинг аутентификации (Auth)

## Story 1: Реализация JWT-валидации в middleware
**Приоритет:** high
**Зависимости:** нет

Як розробник, я хочу реалізувати повноцінну JWT-валідацію в middleware/auth.py замість заглушки, щоб всі запити проходили через єдину точку аутентифікації.

**Acceptance Criteria:**
- Заменить TODO в middleware/auth.py на полноценную JWT-валидацию
- Использовать логику из dependencies.py для decode токена
- Проверять подпись, expiration, извлекать user_id
- Возвращать 401 при невалидном токене

## Story 2: Устранение дублирования кода валидации
**Приоритет:** high  
**Зависимости:** Story 1

Як розробник, я хочу усунути дублювання між middleware/auth.py та dependencies.py, щоб мати єдину логіку JWT-валідації.

**Acceptance Criteria:**
- Вынести общую логику JWT-валидации в utils/jwt_helper.py
- Middleware и dependencies должны использовать одну функцию
- Убрать дублированный код проверки токенов
- Сохранить функциональность admin-проверок и optional auth

## Story 3: Проверка использования JWT-валидации в endpoints
**Приоритет:** medium
**Зависимости:** Story 2

Як розробник, я хочу переконатися що всі endpoints використовують повноцінну JWT-валідацію, щоб забезпечити безпеку API.

**Acceptance Criteria:**
- Проверить все endpoints в api/v2/auth.py
- Убедиться что используется middleware или dependencies для валидации
- Endpoints /login и /verify должны быть без аутентификации
- /me, /refresh, /logout должны требовать валидный JWT

## Story 4: Тестирование рефакторинга аутентификации
**Приоритет:** medium
**Зависимости:** Story 3

Як QA-інженер, я хочу протестувати рефакторинг аутентифікації, щоб переконатися що всі сценарії працюють коректно.

**Acceptance Criteria:**
- Тестировать валидные JWT токены - должны пропускаться
- Тестировать невалидные/expired токены - должны возвращать 401
- Проверить работу protected endpoints
- Убедиться что rate limiting работает
- Проверить admin-функциональность
